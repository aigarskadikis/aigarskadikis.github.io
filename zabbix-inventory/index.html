Zabbix inventory

What is this?

It's possible to utilize invetory fields to store the data (json, txt) of "master item". It's usefull to:
  • Develop "a table" in retro spect. Keep a snippet of most trustable source handy
  • Minimal load (or no load at all) on workstation/server. Enable collector item to populate master data, disable it in next day.
  • Combine Linux/Windows inventory stats together
  • Create a list of gadgets (laptops) at home. Turn on, install agent, shut down, put back in box

CPU model

Windows (native key):
wmi.getall[root\cimv2,SELECT * FROM Win32_Processor]
Linux (native key):
vfs.file.contents[/proc/cpuinfo]

Total memory

Windows (native key):
wmi.getall[root\cimv2,SELECT * FROM Win32_PhysicalMemory]
Linux (native key):
vfs.file.contents[/proc/meminfo]

Size of disk

Windows (native key):
wmi.getall[root\cimv2,SELECT * FROM Win32_DiskDrive]
Linux (native key):
vfs.file.contents[/proc/partitions]

Internal IP address

Linux (native key):
vfs.file.contents[/proc/net/fib_trie]
Windows (native key):
wmi.getall[root\cimv2,SELECT * FROM Win32_NetworkAdapterConfiguration]

OS

wmi.getall[root\cimv2,SELECT * FROM Win32_OperatingSystem]
